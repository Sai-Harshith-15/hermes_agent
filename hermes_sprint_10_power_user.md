# 🚀 Sprint 10: The Power-User Expansion (Final 4 Gaps)

## 1. Executive Summary
With the core priority roadmap at 100% completion, Sprint 10 focuses on the final 4 advanced edge-case features defined in the original Hermes specifications. These features are designed for power-users who want deep programmatic extensibility and low-level system control over the agent.

**Crucial Constraint:** Like Phases 1-4, all logic built here must remain 100% OS-Agnostic, running identically on Windows Docker Desktop setups and local Oracle Linux Ubuntu deployments without triggering `systemd` or pathing errors.

## 2. Implementation Plan by Domain

### 🧩 Gap 1: The Plugin System & SDK (Gap 3)
**The Goal:** Allow community JS bundles dropped into `~/.hermes/plugins/<plugin-name>/dashboard/` to dynamically mount inside the React app without recompiling Vite.

*   **Backend (`backend/app/api/v1/plugins.py`)**
    *   *Static Mount:* In `main.py`, add a new FastAPI mount point specifically for serving the raw JS files dynamically over the single port: 
        `app.mount("/api/plugins-static", StaticFiles(directory=os.path.expanduser("~/.hermes/plugins")), name="plugins")`
    *   *API Route:* `GET /api/v1/plugins/manifests` uses `pathlib` to scan the plugins directory, read each `manifest.json`, and return an array of active plugins and their entry JS file paths.
*   **Frontend (`window.__HERMES_PLUGIN_SDK__`)**
    *   *The SDK:* In `main.tsx` or `App.tsx`, define the global object before React boots so external plugins can tap into your internal dashboard state:
        ```javascript
        window.__HERMES_PLUGIN_SDK__ = {
            addSidebarTab: (id, icon, label, component) => { /* logic to append to UI */ },
            addHeaderWidget: (id, component) => { /* logic to inject into top bar */ }
        };
        ```
    *   *Dynamic Loader:* Create a `PluginLoader.tsx` component that fetches the manifests from the backend and dynamically injects `<script type="module" src="/api/plugins-static/<name>/dashboard/bundle.js">` tags into the document head to execute external UI code safely.

### 🛡️ Gap 2: Advanced Shell Hooks (Gap 12)
**The Goal:** Implement a consent-gated UI to manage precise event triggers and timeouts inside `shell-hooks-allowlist.json`.

*   **Backend (`backend/app/api/v1/hooks.py`)**
    *   *Data Source:* Safely read/write `~/.hermes/shell-hooks-allowlist.json`. (Auto-create the file with an empty `[]` if it doesn't exist).
    *   *CRUD Endpoints:* `GET`, `POST`, `DELETE` to manage hook objects.
    *   *Schema validation:* Enforce payload structure using Pydantic to ensure nothing malformed breaks the agent:
        ```python
        class ShellHook(BaseModel):
            event: str
            command: str
            matcher: str
            timeout: int = 30
        ```
*   **Frontend (`ShellHooksScreen.tsx`)**
    *   A specialized table under the "System Ops" tab.
    *   Include a form to define the `event` (e.g., `on_agent_stuck`), the `command` (e.g., `/usr/bin/python3 alert.py`), and the `timeout`.
    *   Add a prominent **"Consent Granted: Execution Risk Acknowledged"** UI toggle badge to fulfill the "consent-gated" requirement before saving.

### ⏪ Gap 3: Checkpoints Management (Gap 13)
**The Goal:** Provide a UI to view and prune the `/rollback` filesystem, helping users manage disk space natively (especially important for Oracle Free Tier).

*   **Backend (`backend/app/api/v1/checkpoints.py`)**
    *   *Scanner:* `GET /api/v1/ops/checkpoints` uses Python's `os.scandir(os.path.expanduser("~/.hermes/rollback/"))` to list checkpoint files (usually `.zip` or `.db.bak`). Returns the filename, creation date (`os.stat().st_ctime`), and file size (`os.stat().st_size`).
    *   *Pruner:* `DELETE /api/v1/ops/checkpoints/{filename}` safely executes `os.remove()` to delete the target file.
    *   *OS-Safety:* Wraps the directory access in `try/except FileNotFoundError` just in case the `/rollback` directory hasn't been generated by Hermes yet. Protect against `../` path traversal.
*   **Frontend (`CheckpointsScreen.tsx`)**
    *   A sleek list UI displaying snapshots grouped by date inside the Ops dashboard.
    *   Includes a formatting function to display byte sizes as readable MB/GB.
    *   Features a red "Prune Checkpoint" trash icon that dynamically triggers the `DELETE` route to clear disk space.

### 🧹 Gap 4: Skill Curator Status (Gap 14)
**The Goal:** Monitor and pause the background skill maintenance daemon without hard-crashing the container.

*   **Backend (`backend/app/api/v1/curator.py`)**
    *   Since we cannot rely on OS-specific process managers (like `systemctl`), we rely on cross-platform Docker signal management.
    *   *Status:* `GET /api/v1/skills/curator` uses `psutil` to check if a process containing `hermes-curator` is actively running. Returns `{ status: "running" | "paused" }`.
    *   *Toggle:* `POST /api/v1/skills/curator/toggle`. 
        *   To pause: Sends `subprocess.Popen(['pkill', '-STOP', '-f', 'hermes-curator'])` (Sends `SIGSTOP` suspending the process natively in Linux/Docker without losing memory state).
        *   To resume: Sends `subprocess.Popen(['pkill', '-CONT', '-f', 'hermes-curator'])` (Sends `SIGCONT` resuming the process).
*   **Frontend (`SkillsHeader.tsx`)**
    *   Inject a "Curator Daemon: [Active / Paused]" toggle switch into the top right of the Skills Hub UI.

## 3. Docker & Cross-Platform Resilience Rules (Sprint 10)

To ensure this Sprint matches the bulletproof quality of Phases 1-4, rigidly apply these rules:

1. **Path Expansion:** Every single feature in this sprint interacts with the filesystem. You *must* continue wrapping strings in `os.path.expanduser("~/.hermes/...")`. Windows Docker volumes will fail to map paths if `~` is passed literally to the `open()` function.
2. **File Deletion Safety:** When implementing the Checkpoint Pruner, use `pathlib.Path(file).unlink(missing_ok=True)` (Python 3.8+) or wrap `os.remove` in a `try/except FileNotFoundError` to ensure the API doesn't crash if the file was already manually deleted by a user via the CLI.
3. **MIME Types for JS Plugins:** When serving the dynamic plugin JS files via FastAPI's `StaticFiles`, Windows Docker hosts sometimes fail to map the `.js` extension to `application/javascript`. Explicitly initialize the mime types in `main.py` before mounting:
    ```python
    import mimetypes
    mimetypes.add_type('application/javascript', '.js')
    ```
4. **No Dynamic Port Mappings for Plugins:** Because we serve the React app via FastAPI on a single port (`8000`), the `__HERMES_PLUGIN_SDK__` must fetch JS bundles via relative paths (`/api/plugins-static/...`) rather than trying to hit a different domain or port. This guarantees it works flawlessly through your automated Cloudflare Tunnel.

2. **Advanced Shell Hooks:** Strict JSON reading/writing and Pydantic validation.
